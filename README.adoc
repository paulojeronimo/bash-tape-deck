Why I (*{_author}*) Built This {_open-source} Project

*I wanted a way to demonstrate, step by step, the execution of a set of scripts I had written to explain the concept of {AgentSkills}*.
Those scripts were already in good shape and did exactly what I wanted.
The problem was presentation.

I really like asciiart, as you can see above. But I also know there are people who would like to see something more impressive, like this:

Shell scripts are tedious to explain to an audience that does not know much Bash.
I also did not want to use slides, and I definitely did not want to write yet another tutorial document like the many I had already produced before.

What I wanted was something more elegant: an environment where I could put the scripts in motion, let the commands do real work, and still move backward during the presentation to show what had just happened.

That led me to the idea of building an execution environment, using {Docker} or {LXD}, where I could replay the same scripted demonstration as many times as necessary and always return to a known state.

I had already used similar ideas before, but in cloud-hosted environments.
One example was {Katacoda}, which was later discontinued by O'Reilly.

*{BashTapeDeck}* is the local, script-driven version of that idea.
But it can also be executed easily in the cloud through {DevelopmentContainers}.
That means you can run it on practically any cloud environment and, even more easily, on {GitHubCodespaces}.
